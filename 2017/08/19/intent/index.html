Developers之Intent

发表于 2017-08-19 | 分类于 android

Intent是一个你可以请求来自其他应用组件活动的消息对象。通常有以下三种基本的用法：

###Start an activity:
应用中一个activity独占一个屏幕。可以通过给startActivity()方法传递一个携带必要数据的intent对象来start新的Activity。也可以调用startActivityForResult()在目标Activity结束时获取结果，并将结果返回到原Activity中调用原activity中的Onresult()方法。

###Start a service:
service是在后台运行并且提供给用户一个操作接口的组件，可以通过向startService()
传递一个intent来执行一次操作（比如下载文件），如果service是客户—服务器模式
可以通过向bindService()传一个intent去绑定服务。

###Deliver a broadcast：
广播是所有应用可以接受到的一个消息。系统运行时会发出各种广播，比如系统启动
时或者充电的时候。可以通过给sendBroadcast()传递一个intent来向其他应用
发送广播。

##Intent的类型

intent分为显式intent和隐式intent

###显式intent

通过指定类名来开启组件，通常使用显式intent开启自己应用的组件
因为你知道你想启动的活动或者服务的类的名字。

###隐式intent

不明确指明一个类，而是声明一个需要执行的action，允许其他应用
接收并进行处理。例如说，你想要向用户展示地图上的一个位置，可以通过一个
隐式intent去请求开启一个能够显示地图位置的应用。

当你创建一个显式intent来开启一个activity或者service的时候，系统会立即
启动intent指明的应用组件。
当intent是一个隐式intent的时候，Android系统会对比设备上其他APP组件配置
文件中的intent filter，如果intent和一个intent filter相匹配则打开该组件
冰箱其传递intent，如果有多个应用都满足此intent，系统就会显示一个对话框让用户选择
要打开的应用。

intent filter是在应用配置文件中指明自己想要接收的intent的类型；如果不声明任何的
intent filter，应用则只能由显式intent打开。

注意：service通常由显式intent打开，隐式intent可能会有安全风险因为你不知系统打开
的事什么服务，从Android5.0开始，使用隐式intent开启服务系统就会抛出一个异常。

##Building an Intent

intent对象需要携带一些信息去指明将要开启什么组件，比如说组件名称、目录等，另外还
刻意附加一些信息来使组件更好的执行action。

下面是一些基本的信息：
组件名称
这个信息是可选的，但对于显式intent来说是最关键的信息，标志这你想要打开的应用；如果没有
名称这个信息，那么这就是一个隐式intent，通过携带其他的一些信息作为标记，要想打开一个
具体的组件，则必须要指定它的名字。

组件名称必须包括其所在的包名，比如 com.example.ExampleActivity。可以使用setComponent(),
setclass(),setClassName()指定componentname，也可以给intent构造函数参数赋值。

Action
指明希望要执行的行为的一个字符串。

给action赋值可以用setAction()或者用intent构造函数。

定义自己的action需要加一个包名作为前缀：
static final String ACTION_TIMETRAVEL = “com.example.action.TIMETRAVEL”;

Data
指明要通过intent传递的数据对象或者数据类型，数据对象使用一个URI对象来传递，
指定数据类型通过指明MIME type，用来寻找和此intent匹配的组件，设置URL调用
setDdata()方法，设置MIME type调用setType()方法，如果两个都需要指定，则需要
调用setDataAndType()方法；setData和setType两个方法互斥，不能同时调用。

Category
好比目录，将intent分类，或者指明他可以实现怎样的行为，则必须要设置它的
category属性。例如：
CATEGORY_BROWSABLE
表示目标activity允许通过浏览器连接来启动或者展示其组件内容；
CATEGORY_LAUNCHER
应用启动时的初始界面。

以上的一些信息，比如组件名称、action、data和category，是用来系统确定将要启动的
组件；
intent还能携带一些不会其他的信息，这些信息不会影响系统确认目标组件。

Extras

完成action所需要的数据值，可以调用putExtras()方法传参，该方法有两个参数，键值
创建一个Bundle对象，向Bundle传入需要的值，最后将Bundle对象通过putExras()传给
intent。

Flags
用来记录intent的数据属性信息，可以知道系统如何去开启一个活动并且在启动后怎样
处理（例如，判断其是否在最近任务列表中），使用setFlags()设置。

###Example explicit intent
// Executed in an Activity, so ‘this’ is the Context
// The fileUrl is a string URL, such as “http://www.example.com/image.png“
Intent downloadIntent = new Intent(this, DownloadService.class);
downloadIntent.setData(Uri.parse(fileUrl));
startService(downloadIntent);

###Example implicit intent
// Create the text message with a string
Intent sendIntent = new Intent();
sendIntent.setAction(Intent.ACTION_SEND);
sendIntent.putExtra(Intent.EXTRA_TEXT, textMessage);
sendIntent.setType(“text/plain”);

// Verify that the intent will resolve to an activity
if (sendIntent.resolveActivity(getPackageManager()) != null) {
startActivity(sendIntent);
}

##Forcing an app chooser
当有不止一个组件匹配你的intent时，用户可以选择要打开的组件也可
以对action进行默认选择的设置。这是应该向用户展示一个对话框，询
问用户要选择哪一个组件进行开启。
用createChooser()创建一个intent来展示选择对话框，并且将这个
intent传给startActivity()：

Intent sendIntent = new Intent(Intent.ACTION_SEND);
...

// Always use string resources for UI text.
// This says something like "Share this photo with"
String title = getResources().getString(R.string.chooser_title);
// Create intent to show the chooser dialog
Intent chooser = Intent.createChooser(sendIntent, title);

// Verify the original intent will resolve to at least one activity
if (sendIntent.resolveActivity(getPackageManager()) != null) {
    startActivity(chooser);
}

##接收一个隐式intent（intent filter）

为了声明你的应用可以接收处理的隐式intent，必须要在应用配置文件中的每一个组件
加入一个或者多个，每个intent filter指定intent的action、data
和category，只有你的组件的某一个intent filter符合系统消息隐式intent时
系统才会发送intent。
一个应用组件应该为每一个不同的功能单独分离intent filter。例如照片APP应该有
image filter和edit filter。
每个intent filter由元素定义，放置在元素中，在
中，你可以通过指定一下三个元素来说明可以接收的intent：

###action

说明可以响应执行的action

###data

说明处理的数据类型及MIME type

###category

在name属性中声明intent类别，值必须是字符串类型。

注意：为了得到隐式intent，intent filter里必须要包含CATEGORY_DEFAULT category
startActivity()和startActivityForResult()把所有的intent都当做默认intent类。
如果不声明这个类，你的activity将不能响应任何隐式intent。
