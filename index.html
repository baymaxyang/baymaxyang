思考中改变，改变中前行

2017-09-26 life

最近心很乱很烦，也可以说是一直很乱

第一，每天想着要学习精通一门技术，作为以后可能说的发展方向，也希望能够在一年之后的实习然后工作积累一些，方便以后职业道路的发展，大一大二一直觉得大学中在其他方面正常进行的基础上，成绩不需要那么重视，以至于自己一直希望在某个技术领域可以多学一些东西，学着学着到现在发现自己成绩一塌糊涂，想要学习的东西一直也没有办法学好，不知道是自己不够耐心，还是其他的事情太多，没有真正的投入去搞，浪费了时间又没做成事情。

第二，人际关系似乎出现了一些问题，虽然我在其他人眼里算是正常的节奏，有自己的生活态度和目标，因为一些原因，离大家慢慢变远，身边的人有的专注开发，算是不顾成绩，有的在课程学习里认真专研，有的认真上课完成作业但对其他不管不顾，有的似乎从不考虑未来的发展，不挂科就是追求，其他时间就是玩乐；觉得自己无助，没有人告诉现阶段的我该做什么,什么样的路才是最合适的，自己选择的会不会从始至终就是一个错误。

第三，和前文相关，因为自己家庭现况，想要能够在未来工作中能够有一定优势，当然也是自己想要有一些成就和对于这个专业方向的兴趣，可自己过于焦躁，一直想在技术上有一定成就，今天想学这个，明天想学那个，最后什么都没学会，什么都只是入门了解；自己付出的精力似乎不比别人少，却感觉一无所获，可能就是要在失败中成长吧，希望我 能够不断从迷茫中开辟出道路，在仅剩的一年多时间里丰富自我，给未来的自己更多的期待。

Developers之Intent

2017-08-19 android

Intent是一个你可以请求来自其他应用组件活动的消息对象。通常有以下三种基本的用法：

Start an activity:

应用中一个activity独占一个屏幕。可以通过给startActivity()方法传递一个携带必要数据的intent对象来start新的Activity。也可以调用startActivityForResult()在目标Activity结束时获取结果，并将结果返回到原Activity中调用原activity中的Onresult()方法。

Start a service:

service是在后台运行并且提供给用户一个操作接口的组件，可以通过向startService()传递一个intent来执行一次操作（比如下载文件），如果service是客户—服务器模式可以通过向bindService()传一个intent去绑定服务。

Deliver a broadcast：

广播是所有应用可以接受到的一个消息。系统运行时会发出各种广播，比如系统启动时或者充电的时候。可以通过给sendBroadcast()传递一个intent来向其他应用发送广播。

Intent的类型

intent分为显式intent和隐式intent

显式intent

通过指定类名来开启组件，通常使用显式intent开启自己应用的组件因为你知道你想启动的活动或者服务的类的名字。

隐式intent

不明确指明一个类，而是声明一个需要执行的action，允许其他应用接收并进行处理。例如说，你想要向用户展示地图上的一个位置，可以通过一个隐式intent去请求开启一个能够显示地图位置的应用。

当你创建一个显式intent来开启一个activity或者service的时候，系统会立即启动intent指明的应用组件。
当intent是一个隐式intent的时候，Android系统会对比设备上其他APP组件配置文件中的intent filter，如果intent和一个intent filter相匹配则打开该组件冰箱其传递intent，如果有多个应用都满足此intent，系统就会显示一个对话框让用户选择
要打开的应用。

intent filter是在应用配置文件中指明自己想要接收的intent的类型；如果不声明任何的intent filter，应用则只能由显式intent打开。

注意：service通常由显式intent打开，隐式intent可能会有安全风险因为你不知系统打开的事什么服务，从Android5.0开始，使用隐式intent开启服务系统就会抛出一个异常。

Building an Intent

intent对象需要携带一些信息去指明将要开启什么组件，比如说组件名称、目录等，另外还刻意附加一些信息来使组件更好的执行action。

下面是一些基本的信息：
组件名称
这个信息是可选的，但对于显式intent来说是最关键的信息，标志这你想要打开的应用；如果没有名称这个信息，那么这就是一个隐式intent，通过携带其他的一些信息作为标记，要想打开一个具体的组件，则必须要指定它的名字。

组件名称必须包括其所在的包名，比如 com.example.ExampleActivity。可以使用setComponent(),setclass(),setClassName()指定componentname，也可以给intent构造函数参数赋值。

Action
指明希望要执行的行为的一个字符串。

给action赋值可以用setAction()或者用intent构造函数。

定义自己的action需要加一个包名作为前缀：
static final String ACTION_TIMETRAVEL = “com.example.action.TIMETRAVEL”;

Data
指明要通过intent传递的数据对象或者数据类型，数据对象使用一个URI对象来传递，指定数据类型通过指明MIME type，用来寻找和此intent匹配的组件，设置URL调用setDdata()方法，设置MIME type调用setType()方法，如果两个都需要指定，则需要调用setDataAndType()方法；setData和setType两个方法互斥，不能同时调用。

Category
好比目录，将intent分类，或者指明他可以实现怎样的行为，则必须要设置它的category属性。例如：
CATEGORY_BROWSABLE
表示目标activity允许通过浏览器连接来启动或者展示其组件内容；
CATEGORY_LAUNCHER
应用启动时的初始界面。

以上的一些信息，比如组件名称、action、data和category，是用来系统确定将要启动的组件；
intent还能携带一些不会其他的信息，这些信息不会影响系统确认目标组件。

Extras

完成action所需要的数据值，可以调用putExtras()方法传参，该方法有两个参数，键值创建一个Bundle对象，向Bundle传入需要的值，最后将Bundle对象通过putExras()传给intent。

Flags
用来记录intent的数据属性信息，可以知道系统如何去开启一个活动并且在启动后怎样处理（例如，判断其是否在最近任务列表中），使用setFlags()设置。

接收一个隐式intent（intent filter）

为了声明你的应用可以接收处理的隐式intent，必须要在应用配置文件中的每一个组件加入一个或者多个，每个intent filter指定intent的action、data和category，只有你的组件的某一个intent filter符合系统消息隐式intent时
系统才会发送intent。一个应用组件应该为每一个不同的功能单独分离intent filter。例如照片APP应该有image filter和edit filter。每个intent filter由元素定义，放置在元素中，在中，你可以通过指定一下三个元素来说明可以接收的intent：

action

说明可以响应执行的action

data

说明处理的数据类型及MIME type

category

在name属性中声明intent类别，值必须是字符串类型。

注意：为了得到隐式intent，intent filter里必须要包含CATEGORY_DEFAULT categorystartActivity()和startActivityForResult()把所有的intent都当做默认intent类。如果不声明这个类，你的activity将不能响应任何隐式intent。

欢迎来到Baymaxyang的博客

2017-08-17 about me
先说一下博客名字Baymaxyang的由来，Baymax大家都知道，中文名字叫大白，是电影超能陆战队里的一个憨厚、呆萌的卡通形象，呆萌不适合我，我比较喜欢他的憨厚、正直、真诚；另外自己还是一名学生，Android开发刚刚入门，是一名程序员小白，希望通过自己的努力成长，大白是我心里的高级工程师；yang想必都猜得到，是我姓的一个标志，也是模仿Android界一位大神在网名后加自己的姓；故最后有了Baymaxyang的产生，又去万网注册了www.bayamxyang.com的域名，就是我博客的域名，博客网站是使用hexo在github上搭建的静态网站，绑定了自己注册的域名，之后有时间我会发一篇Hexo和github搭建个人博客以及如何绑定域名的文章。

搭建自己博客的目的，Android学习中会遇到很多新的东西，学习了之后就会去学习其他的知识，由于原来的知识没有经过深度的学习，首先会造成对知识点不够理解，其次是不会应用，只能大概了解。于是想通过对自己学习过程中的知识点做一个记录，便于深刻理解，也作为以后复习的资料；最近会针对Android Developers中的开发文档进行学习，也会不定期进行一些学习记录，记录Android开发中的一些重点。另外，这个博客也不仅仅是自己的技术博客，也希望是监督我阅读和写作的一个工具，自己觉得需要记录的心得还有自己对身边一些时事都会在这里更新；最近刚刚有一个小想法，这个博客也可以作为我的网络便签，因为发现生活中经常会遇见一些要记录的事物或偶尔在快速阅读中发现自己想要收藏的东西，但收藏之后就不知所踪，也不知道是在哪，什么时候收藏的什么东西，感觉错过了很多美好的东西。

现在自己是中国矿业大学的一名本科生，专业是当下热门的计算机，大学中难的是找到自己真正感兴趣，想要长久发展下去的领域；虽说是确定了计算机行业，但其中的分支又甚是繁多，各个分支之间是相同的，但是想要必须要做到在某一个方面做到专业水平；高中的时候喜欢电子电路，喜欢集成电路，原因是自己喜欢一些科技圈的一些智能硬件，想要开发出自己在生活中发现的idea，所以选择了计算机，希望能够在嵌入式方向深入学习；大一大二用了一年半也一直在自学单片机嵌入式，后来发现自己离自己最初的目标有一定距离，一个产品不是靠想象、靠一个人的设计就能做出来的，而且以后参加工作自己做公司给定的项目，自己能够做的事情也很局限，再三考虑后放弃了这一年多的学习和两年的目标。对我来说，计算机行业就分为两个区域，一个是后端数据库设计、架构设计、内存优化等，如果计算机领域是一个黑箱，那么这个区域就是黑箱中的黑箱，另一区域就是用户可以直接接触到的东西，别如说移动应用开发、前端、嵌入式等；大家都知道后端要涉及的知识要比前端更深、要学习的东西也更难，其实我觉得只要你是真的热爱一个领域，花时间去钻研，每个领域之间就没有实质性的区别。

鉴于个人比较喜欢生活中的设计，对后端那些东西不是很感冒，觉得唯有设计出实用、新颖的东西更有成就感，故最后选择了Android移动应用开发；第一是Android在市场中的份额，第二是Android相对于IOS的开源性，第三是Android应用每个人都可以接触到，虽然说近一年显示微信小程序和直达应用的兴起，但和应用程序相比，流畅度和使用体验都要差一大截，目前来说Android开发市场前景还很好。
